# 调试记录 — 2026-05-17

## Bug 1: MergedCell 导出崩溃 (已修复)

**症状**: 点击「全部输出」→ `ERROR Export failed: 'MergedCell' object attribute 'value' is read-only`

**根因**: `output_writer.py` 汇总段两处使用了变量 `ws`（指向主循环最后创建的构筑物 sheet）而非 `ws_sum`（汇总 sheet）。当汇总数据行号与最后那个 sheet 的合并单元格行号重叠时，`ws.cell(row, col, value=...)` 命中 `MergedCell` → crash。

**修复**:
- `output_writer.py:311-316` — 各构筑物概况数据: `ws.xxx` → `ws_sum.xxx`
- `output_writer.py:328-333` — 输出日志数据: `ws.xxx` → `ws_sum.xxx`
- `output_writer.py:_write_section()` — 边框循环只设 col=1，避免触碰 MergedCell

**教训**: openpyxl 3.1.5 中 `MergedCell.__slots__` 不含 `value`，任何 `merged_cell.value = x` 都抛 `AttributeError`。合并区域只会把左上角单元格保留为普通 Cell，其余列全变成 MergedCell。写入数据时若目标行恰好是合并标题行 → 必崩。

## Bug 2: 工程概算数据源脱节 (已修复)

**症状**: 调节池/CASS/高密度沉淀池报 "无有效计算结果，跳过概算"，但节点绿灯正常、UI面板显示正常、Excel导出正常。

**根因**: 
```
_on_export_cost() → execute() → results dict (丢弃!)
                 → estimator.estimate(executor) → 读 executor._nodes[nid]._result
```
`execute()` 返回的 `results` dict 是 UI 状态灯和 Excel 导出的权威数据源，但 `CostEstimator` 绕过了它，直接读 `node._result`（可能为 None 或过期）。

**为什么偶发**: `execute(force_all=False)` 第二次调用时，CLEAN 节点的 `node.result` 检查 (`if node.result:`) 如果 `_result` 为 None → 跳过 → 不在 results 中 → 但状态灯没更新（保留上次 F5 的绿色）。用户看到绿灯但 estimator 看到空结果。

**修复** (4 文件):
- `cost_estimator.py`: `estimate()` 接受 `results` 参数，`_structure_civil()` 接受 `result` 参数，优先使用传入结果
- `report_writer.py`: `write_cost_report()` 和 `_write_structure_sheet()` 接受 `results`，尺寸提取时优先使用
- `main_window.py`: `_on_export_cost()` 把 `execute()` 返回的 `results` 传入 estimator 和 report_writer
- `tests/test_cost_estimator.py`: 新增 16 个 TDD 测试

**核心教训**: 
- **永远不要在消费端直接读 `node._result`** — 它是缓存，可能过期/缺失
- **`execute()` 返回的 `results` dict 是唯一权威数据源** — UI、导出、概算全部从这里取
- 字典统一后的数据流: `execute() → results → {UI, export, cost}`

## 更新后的数据流架构

```
execute()  →  results: Dict[str, NodeResult]  ← 唯一权威源
    │
    ├── _on_calc_rest()     → UI 状态灯 + 水质面板
    ├── _on_export_all()    → write_classified_output(results)
    └── _on_export_cost()   → estimator.estimate(results=results)
                            → write_cost_report(results=results)

node._result  ← 仅由 node.execute() / from_dict() 写入
              ← 仅由 execute() 内部读取 (构建 results dict)
```

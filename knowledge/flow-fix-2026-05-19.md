# 流量传输修复 — 2026-05-19

## 背景

用户报告三个问题：
1. "水量以用户输入为准" — 水量传输未正确使用用户设定值
2. "水质与水量计算都报废了" — 清洁路径零值传播导致全链报废
3. "流量无法向下传输，面板显示始终为570" — CombinerNode 面板停滞

## 根因链

### Bug 1: GraphExecutor 仅识别 InputNode
- `graph_executor.py:225`: `isinstance(node, InputNode)` 只处理一种输入节点
- KwInputNode、PipeNetworkNode 未被识别 → 走 `_merge_upstream` → 孤儿节点获默认值

### Bug 2: 清洁路径零值传播 (自引入)
- 流量源节点用零值占位 → 清洁路径用零值计算下游数据 → upstream_data 存入零值
- 影响: InputNode/KwInputNode/PipeNetworkNode/WaterQualityNode 全部归零

### Bug 3: _on_calc_all 过早 CLEAN 标记 (关键!)
- `_on_calc_all()` 先直接调用 `pipe_node.execute()` → 标记 CLEAN
- GraphExecutor 后续运行 → 无法检测 PipeNetworkNode 变更 → CombinerNode 走清洁路径
- 清洁路径维度不更新 → **面板始终显示 570 L/s (默认 0.57 m³/s)**

```
F5 键 → _on_calc_all()
  → pipe_node.execute()  # 标记 CLEAN ❌
  → _on_calc_rest()
    → executor.execute(force_all=False)
    → PipeNetworkNode: CLEAN (刚被标记)
    → CombinerNode: CLEAN, 无上游 DIRTY → 清洁路径
    → 维度不更新 → 始终 570!
```

## 修复方案 (6 文件)

### 1. input_node.py — 添加 execute() 重写
```python
def execute(self, flow, quality):
    # 从用户参数构建 WaterFlow，上游流量不存在
    out_flow = WaterFlow(Q_design=Qd, Q_avg_daily=Qad, Kz=Kz)
    downstream_quality = WQ(BOD5=self.water_quality.BOD5, ...)
    return result, out_flow, downstream_quality
```
与 KwInputNode/PipeNetworkNode/WaterQualityNode 保持一致模式。

### 2. graph_executor.py — 端口驱动流量源检测 (取代 isinstance)
```python
is_flow_source = not any(WATER|MIXED in input_ports) and any(WATER|MIXED in output_ports)
```
自动覆盖: InputNode, KwInputNode, PipeNetworkNode，及未来任何输入节点。

### 3. graph_executor.py — 强制无 WATER/MIXED 输入节点执行
```python
needs_execution = nid in dirty_set or force_all or not any(WATER|MIXED in input_ports)
```
防止流量源/水质源节点走清洁路径（零值占位符会污染下游）。

### 4. graph_executor.py — _merge_upstream 孤儿节点零值
```python
# 旧: WaterFlow() (Q_design=0.57 硬编码) → 静默错误
# 新: WaterFlow(Q_design=0.0, ...) → 立即暴露问题
```

### 5. graph_executor.py — 清洁路径流量维度同步
```python
# 清洁路径新增: 同步更新 m³/s, L/s, m³/d, m³/h 维度
for dim_name in list(node.result.dimensions.keys()):
    if unit == "m³/s" and "设计流量" in dim_name:
        node.result.dimensions[dim_name] = (input_flow.Q_design, unit)
```

### 6. main_window.py — _on_calc_all 移除过早 pipe_node.execute()
```python
# 旧: pipe_node.execute() 直接调用 → 标记 CLEAN → GraphExecutor 无法传播
# 新: 全部交由 GraphExecutor 统一管理
def _on_calc_all(self):
    self._on_calc_rest()
```

## 架构原则

1. **流量源节点 = execute() 重写** — InputNode/KwInputNode/PipeNetworkNode 全部重写
2. **端口驱动检测 > isinstance** — 未来新增输入节点无需修改 GraphExecutor
3. **execute() 路径 > 清洁路径** — 无 WATER/MIXED 输入的节点强制走 execute()
4. **GraphExecutor 是唯一调度者** — 禁止外部直接调用节点 execute() 后依赖 GraphExecutor 继续
5. **零值 > 默认值** — 孤儿节点返回零值立即暴露问题，而非静默错误

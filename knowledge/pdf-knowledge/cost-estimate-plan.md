# 工程概算模块 — 设计与实现计划

## 一、概算模型

### 1.1 费用组成

```
工程总造价 = 建筑工程费 + 设备购置费 + 安装工程费 + 其他费用 + 预备费
```

| 费用类别 | 占比(估算) | 计算方式 |
|---------|-----------|---------|
| 建筑工程费 | 50-60% | 按构筑物尺寸 × 综合单价 |
| 设备购置费 | 20-30% | 设备清单 × 设备单价 |
| 安装工程费 | 8-12% | 设备购置费 × 安装费率 |
| 其他费用 | 5-8% | (建+设+安) × 费率 |
| 预备费 | 5-8% | (建+设+安+其他) × 费率 |

### 1.2 构筑物造价估算

每个处理单元从 `NodeResult.dimensions` 提取：

| 构筑物 | 提取维度 | 单价（参考2019黑龙江定额） |
|--------|---------|--------------------------|
| 调节池 | 混凝土量估算(m³) | 850 元/m³ (C30, 含模板钢筋) |
| 格栅间 | 栅槽尺寸 B×L×H | 1200 元/m³ (含设备基础) |
| 沉砂池 | 混凝土量/总高度 | 900 元/m³ (圆形池体) |
| 初沉池 | 混凝土量/总高度 | 850 元/m³ (圆形池体) |
| CASS | 总容积×1.2(壁厚系数) | 800 元/m³ (矩形池体) |
| 高密度沉淀池 | 池面积×总高×1.15 | 900 元/m³ |
| V型滤池 | 池面积×总高×1.2 | 1000 元/m³ (含滤板) |
| 紫外消毒 | 渠道尺寸 | 2000 元/m (含设备) |

## 二、实现方案

### 2.1 数据结构

```python
@dataclass
class CostItem:
    name: str          # 费用名称
    category: str      # 建筑工程/设备购置/安装工程/其他
    quantity: float    # 工程量
    unit: str          # 单位
    unit_price: float  # 单价（元）
    total: float       # 合价（元）

@dataclass
class CostEstimate:
    project_name: str
    items: List[CostItem]
    civil_cost: float     # 建筑工程费
    equipment_cost: float # 设备购置费
    install_cost: float   # 安装工程费
    other_cost: float     # 其他费用
    contingency: float    # 预备费
    total_cost: float     # 总造价
```

### 2.2 输出格式

Excel 报告包含 sheet:
1. **编制说明** — 工程概况、编制依据
2. **工程总造价汇总表** — 分类汇总
3. **建筑工程费明细** — 各构筑物分部分项
4. **设备购置费明细** — 设备清单
5. **管网工程造价** — 管道分项
6. **单位工程造价指标** — 元/m³ 处理能力

### 2.3 文件结构

```
ddesign_tool/src/models/cost/
    __init__.py
    cost_estimator.py     # 核心概算引擎
    unit_prices.py        # 单价数据库
    report_writer.py      # Excel 报告生成
```

## 三、实施步骤

1. 创建 `unit_prices.py` — 单价数据库（土建/设备/管道）
2. 创建 `cost_estimator.py` — 遍历 GraphExecutor 结果，计算各构筑物造价
3. 创建 `report_writer.py` — 生成规范格式 Excel 报告
4. UI 中增加"概算"按钮 → 生成报告并打开

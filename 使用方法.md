## 2. 快速上手

### 2.1 新建项目

1. `Ctrl+N` 新建，或打开 `yyx.ddesign.json` 示例项目
2. **"➕ 添加节点"** → 按处理阶段选择构筑物
3. **右键按住输出端口**（彩色圆点）→ 拖到目标输入端口 → 连线

### 2.2 标准连线

```
管网输入 ──→ 进水水质 ──→ 合并 ──→ 调节池 ──→ ... ──→ 紫外消毒池
```

| 端口颜色 | 类型 | 传递内容 |
|---------|------|---------|
| 🔵 蓝色 | WATER | 水量 |
| 🟢 绿色 | QUALITY | 水质 |
| 🟠 橙色 | MIXED | 水量+水质 |
| 🟣 紫色 | SLUDGE | 污泥流 |

### 2.3 计算

- **F5** → 全流程 DAG 计算
- 节点右下角状态灯: 🟢=完成 🟡=需重算 🔴=失败
- 方案浏览器会自动同步当前节点的结果

## 3. 方案浏览器（默认模式）

选中水处理节点后，右侧自动显示方案浏览器。

### 3.1 工作原理

1. 根据 `discretization.json` 枚举所有参数组合
2. 向量化批量计算 → 约束过滤 → 按安全裕度排序
3. 当前已应用的方案显示 **绿色 ✓** 并自动选中

### 3.2 操作

| 操作 | 效果 |
|------|------|
| 点击方案行 | 查看详情（尺寸、约束、成本） |
| **▶ 应用选中方案** | 参数写入节点，结果面板同步更新 |
| **★ 推荐方案** | 跳转到安全裕度最高的方案 |
| 💰/📐 切换排序 | 按成本 / 按安全裕度排列 |
| 筛选下拉框 | 按参数值过滤方案列表 |

### 3.3 约束面板

约束面板显示每个约束的当前值和限值：
- 绿色 ✅ = 通过
- 红色 ❌ = 不通过
- 限值可临时修改（仅影响当前会话的方案筛选）

## 4. 手动微调模式

切换到「🎚 手动微调」后：

- **参数滑块** + 数值输入框 — 调整设计参数
- **去除率滑块** — 调整 6 项污染物去除率（0~100%）
- 修改后按 F5 重算

```
格栅(栅渣) + 沉砂池(沉砂) + 初沉池 + CASS(剩余污泥) + 高密池(化学污泥)
    → 合并 → 浓缩池(96%→94%) → 消化池(厌氧) → 脱水间(→78%) → 干化
```

## 9. 常见问题

**Q: 方案浏览器和结果面板数字不一样？**
A: 方案浏览器打开时会自动选中并应用当前参数对应的方案。如果当前参数不在可行空间内，会自动应用推荐方案。F5 后浏览器也会自动同步。

**Q: 修改参数后方案没变？**
A: 按 F5 重新计算。方案浏览器会在 F5 后自动重新加载。

**Q: 工程概算为 0？**
A: 检查模组的 `_vectorized_compute` 是否输出了 `concrete_m3` 字段。

**Q: Excel 输出中构筑物尺寸 section 为空？**
A: v4.5 已修复。此前方案浏览器应用的向量化解无法被正确分类为物理尺寸。新模组必须创建 `labels.json`。

**Q: 出水管水头损失等变量在 Excel 中看不到？**
A: v4.5 已修复。此前 Excel 过滤器误将以"出水"开头的管路水力变量当作水质数据排除。现在 UI 和 Excel 共用 `split_dimensions()` 统一过滤。

**Q: 矿井水处理模组无可行方案？**
A: v4.5 已全面修复。4 个矿井水模组（调节池、沉砂池、混凝反应池、磁分离）均通过增加构筑物数量或修正不合理约束恢复正常。

## 7. Mod Validator 验证器 (v5.0+)

### 7.1 CLI 模式

```bash
# 验证所有模组 (快速模式)
ddesign_tool.exe --validate --all

# 深度验证 (含计算烟雾测试和向量化对标量)
ddesign_tool.exe --validate --all --deep

# 验证指定模组
ddesign_tool.exe --validate --mod=tiaojiechi

# CI 模式 (有 FAIL/ERROR 返回非零)
ddesign_tool.exe --validate --all --deep --ci

# 生成 HTML 报告
ddesign_tool.exe --validate --all --output=report.html

# 基线管理
ddesign_tool.exe --validate --generate-baseline  # 生成基线
ddesign_tool.exe --validate --all --baseline      # 使用基线抑制已知问题
```

### 7.2 GUI 内启动

菜单栏 → 🔧 工具 → 模组验证器 (快速/深度)

### 7.3 检查项说明

| 检查项 | 说明 |
|--------|------|
| 配置完整性 | 验证 discretization.json 中的 free/fixed/constraint 字段 |
| 计算烟雾测试 | 用默认参数运行 calculate()，检查是否返回有效结果 |
| 约束一致性 | 验证 constraint_keys 与向量化输出的 ok_*/val_* 字段匹配 |
| 向量化对标量 | 对比向量化和标量计算的关键尺寸一致性 |
| UI 契约 | 验证 ParamDef 定义与 discretization.json 参数范围匹配 |

## 8. 约束面板联动 (v5.0+)

约束面板中的限值与方案浏览器的筛选逻辑联动修改。修改约束限值后，方案空间会动态重新过滤。

**跨类别参数同步**: 修改 fixed 参数（如 HRT）时，对应的约束限值（如"实际 HRT"）会自动同步调整。

---

## 9. 基线系统 (v5.0+)

基线系统用于管理已知的验证器警告/失败项。

- `.validator-baseline.json`: 自动生成，记录当前所有 FAIL/WARN 的指纹
- `.validator-notes.json`: 手动编写，记录有意偏离的设计决策说明

生成基线后运行 `--baseline` 模式，已验证的已知问题会被抑制，仅显示新增问题。
